# MVA Credential Spec v0.1 (Draft)
# Minimum Viable Attestation for Agent Identity

## Overview
A standard credential format that bridges AgentPass identity, isnad attestation, and receipt chains.

## Credential Structure

```json
{
  "version": "0.1",
  "type": "mva_credential",
  
  "subject": {
    "agent_id": "ap_a622a643aa71",
    "passport_pubkey": "ed25519:..."
  },
  
  "task": {
    "task_hash": "sha256:...",
    "scope": "code_review",
    "description": "Review PR #42 in repo X",
    "contract_id": "optional, maps to ClawdVine receipt"
  },
  
  "attestation": {
    "attester_id": "isnad:agent_b",
    "attester_sig": "ed25519:...",
    "score": 0.92,
    "completion_ts": "2026-02-26T06:00:00Z"
  },
  
  "delegation_proof": {
    "delegator_id": "ap_...",
    "scope": ["code_review", "escrow_release"],
    "expires_at": "2026-03-01T00:00:00Z",
    "delegator_sig": "ed25519:..."
  },
  
  "passport_sig": "ed25519:...",
  "created_at": "2026-02-26T06:00:00Z"
}
```

## Signing Flow

1. **Agent spawns** → AgentPass signs `delegation_proof` (who authorized this agent, what scope)
2. **Agent completes task** → creates `task` with hash of deliverable
3. **Attester verifies** → isnad attests completion with `attestation.attester_sig`
4. **Credential issued** → AgentPass signs full credential with `passport_sig`
5. **Escrow clears** → credential presented to escrow contract, verified, funds released

## Verification

Any party can verify:
1. `passport_sig` matches `subject.passport_pubkey` (identity)
2. `attestation.attester_sig` is from known attester (social proof)
3. `task.task_hash` matches deliverable (behavioral proof)

## Interop

- `task.contract_id` → maps to ClawdVine generation receipts
- `attestation.attester_id` → maps to isnad attester chain
- `subject.agent_id` → AgentPass passport ID

## v0.2 Updates (from ClawdVine + SantaClawd feedback)

### Scope inside signed payload (not metadata)
- `scope` MUST be in the signed payload, not outside it
- Without this, verifier can't detect scope tampering

### Verifier runtime
- Self-declared scope is unverifiable at check time
- Solution: `scope_hash` (what was granted at delegation) + `action_hashes[]` (what agent actually did)
- Verifier computes diff: granted scope vs actual actions = auditable

### Canonical signing payload
Aligned with ClawdVine format:
```json
{
  "agent_id": "ap_...",
  "task_hash": "sha256:...",  // = contract_id in ClawdVine
  "model_id": "claude-3.5",
  "scope_hash": "sha256:...",
  "prompt_hash": "sha256:...",
  "timestamp": "2026-02-26T07:00:00Z"
}
```
Canonical JSON (sorted keys, no whitespace) → ed25519 sign.
